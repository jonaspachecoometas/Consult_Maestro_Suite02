## 3. Rotas sem proteção de tenant — 249 identificadas

**Definição:** rota com `isAuthenticated` mas SEM `requireTenant`/`requireSuperadmin` E SEM `checkTenantAccess` inline.

### 3A — Módulos com maior exposição

**CRM — 40 rotas desprotegidas** (maior risco por volume e sensibilidade):
- Proposals, Contracts, Milestones, Activities, Pipeline stages
- Partners e Commissions (sem qualquer filtro de tenant visível)
- Exemplo: `GET /api/crm/contracts` — qualquer usuário autenticado pode ver contratos de outros tenants se a rota não filtra

**Scrum — 26 rotas desprotegidas:**
- PATCH/DELETE de sprints, backlog items, timesheets, team members
- Risco: usuário de Tenant A pode alterar sprint de Tenant B se souber o ID

**Processos — 19 rotas desprotegidas:**
- PATCH/DELETE de processos, etapas, diagramas, KPIs, recomendações
- GETs de etapas e diagramas sem checagem de tenant do processo pai

**Suporte — 14 rotas desprotegidas:**
- Tickets, types, comentários
- `GET /api/support/tickets` — sem filtro de tenant visível

**Tenants — 13 rotas desprotegidas** (risco diferente: exposição da estrutura):
- `GET /api/tenants` — retorna lista de todos os tenants? Precisa verificar
- `GET /api/tenants/:id` — qualquer usuário autenticado lê dados de qualquer tenant?
- `POST /api/tenants`, `DELETE /api/tenants/:id` — sem requireSuperadmin?

**Knowledge/Brain — 17 rotas desprotegidas:**
- `GET /api/brain/items` — retorna items de todos os tenants se não filtrar
- CRUD de categorias e artigos

### 3B — Rotas de alto risco individual

```
GET  /api/collaborators              — lista colaboradores sem filtro tenant
GET  /api/collaborators/:id         — qualquer ID é acessível
PATCH /api/collaborators/:id        — mutação sem verificar tenant owner
```

### 3C — Rotas provavelmente OK mas com proteção frágil

As 61 rotas que usam `checkTenantAccess` inline estão funcionalmente protegidas, mas o padrão é frágil: se o registro não tem `tenantId` (pois é nullable), `checkTenantAccess` retorna `false` para todos os tenants — inclusive o dono legítimo.

```typescript
// Em checkTenantAccess:
if (resource.tenantId === null || resource.tenantId === undefined) return false;
// Isso significa: registro sem tenant_id = inacessível por todos (exceto superadmin)
// Silencioso — não é um erro, simplesmente retorna 403
```
