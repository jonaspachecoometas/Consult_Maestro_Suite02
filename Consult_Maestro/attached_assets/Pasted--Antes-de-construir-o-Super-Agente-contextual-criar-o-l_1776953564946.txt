O que o Replit Agent precisa fazer
É a Opção C da Fase 1 do doc Multi-Provider AI — só o schema e a UI de configuração, sem migrar os agentes ainda:
1. Schema: tabela tenant_ai_configs
   → tenantId, provider, apiKeyEnc (criptografada), model, isActive
   → Usar o cryptoService existente para criptografar

2. Rotas:
   → GET  /api/ai/config  (lista configs sem expor a key)
   → POST /api/ai/config  (salvar/atualizar)
   → POST /api/ai/config/test (testar conexão)

3. UI: aba "IA & Modelos" em Minha Empresa
   → Card por provider
   → Campo de API Key (type=password, nunca mostra o valor salvo)
   → Botão Testar conexão
   → Badge Conectado / Usando padrão da plataforma
Isso desbloqueia o Super Agente contextual usar a chave do tenant, e já abre as portas para o multi-provider completo depois.
