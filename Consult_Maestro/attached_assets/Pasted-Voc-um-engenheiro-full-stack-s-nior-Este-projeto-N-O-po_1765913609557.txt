Você é um engenheiro full-stack sênior. Este projeto NÃO pode depender do Replit fora do ambiente do Replit. O Replit será usado apenas como IDE/ambiente de desenvolvimento, mas o sistema deve rodar em qualquer servidor (Docker/VM/VPS) usando apenas ENV vars e Postgres.

Objetivo

Implementar autenticação dual-mode e portável:

OIDC Auth0 (login social/SSO via Authorization Code Flow).

Auth local (“puro”) com e-mail + senha no Postgres.

Ambos devem alimentar o mesmo mecanismo de sessão (cookie-based session) no backend.

Proibições (crítico)

Não usar Replit Auth (nem replit-auth, @replit/*, headers específicos, REPL_ID como requisito, callbacks dependentes do Replit, ou qualquer SDK exclusivo).

Não usar URLs hardcoded do Replit. Tudo deve usar APP_BASE_URL e OIDC_CALLBACK_PATH.

Não assumir que o app sempre roda por trás do proxy do Replit.

Requisitos técnicos (portabilidade)

Back-end deve montar redirect_uri sempre como: ${APP_BASE_URL}${OIDC_CALLBACK_PATH}.

Respeitar trust proxy quando necessário (para cookies secure atrás de proxy), mas sem depender do Replit.

Cookies de sessão:

httpOnly: true

secure: true em produção

sameSite compatível com OIDC (use configuração segura que funcione com redirect)

CORS e cookies devem funcionar com o domínio real definido em APP_BASE_URL.

Endpoints mínimos

OIDC (Auth0):

GET /api/login/oidc inicia fluxo OIDC

GET /api/callback trata retorno, valida, cria sessão e upsert do usuário

POST /api/logout limpa sessão e, se aplicável, redireciona para logout do Auth0

Local (“puro”):

POST /api/auth/register

POST /api/auth/login

GET /api/auth/me

POST /api/logout (mesmo endpoint acima)
